


	为了减少apk体积和加快gradle编译速度，比起依赖module，引用aar更为友好。
	
	使用Terminal命令 gradlew 模块名:clean 以及 gradlew 模块名:assemble,会在
	模块名/build/outputs/aar文件夹下生成aar文件。
	或者直接运行一下项目也可以在上述文件夹生成aar文件。
	
	
	
	
	那么如何在library模块中正确引用aar？
	
	
	
	1、在application所在模块的build.gradle文件中allprojects节点加入如下一段：
	
		repositories {
			flatDir {
				dirs 'libs', '../模块名/libs', '../模块名/libs'
			}
		}
		
		多个library模块引用aar用逗号隔开。
		
		
	2、将aar文件复制到library模块下的libs文件夹；
		
		
	3、然后在library模块的build.gradle文件中还要正确引入：
	
		在android节点下添加：
		
		repositories {
			flatDir {
				dirs 'libs'
			}
		}
		
		dependencies节点：
		
		dependencies {
			compile fileTree(include: ['*.jar'], dir: 'libs')
			compile(name: 'aar名字', ext: 'aar')
		}
		
		
	*有一个需要特别注意的地方：
	
		*引用aar的library模块需要在build.gradle文件添加依赖原aar模块依赖的所有库；
		
		*引用的aar本身有引用其他aar的只需要在library模块的build.gradle文件中同样引用其他aar，
		不需要复制其他aar文件到library的libs文件夹下；
